InfluxDB

A promising open-source, distributed, time-series database*
useful for recording metrics, events and performing analytics.
Based on LevelDB, written by Google.

* Designed to address the problem of storing data resulting from
successive measurements made over a period of time.

Use cases and benefits

  1. Successive metrics
  2. Real-time monitoring
  3. Historical analysis
  4. Implementation speed detection (POC)

Features

  1. Fast
  2. Key-value pairs data model
  3. Schemaless, tags
  4. Built-in HTTP API
  5. InfluxQL

Pros

  1. Real-time
  2. Data-type rejection
  3. Evolution cadence
  4. Cluster (alpha)

Setup

  1. Easy installation
  2. No external dependencies
  3. Configuration file
  4. Hardware recommendation
    • 2 SSD volumes
    • 8G RAM each machine
    • Best performance using C3 class of machines

Networking

  1. Admin Panel (port 8083)
  2. Client/server communication (port 8086)
  3. Clustered instances (ports 8088 and 8091)

CLI

			# influx

			> CREATE DATABASE db1

			Using database db1

Data Model

			> INSERT cpu_load_short,host=server01,region=us-west value=0.64 1434055562000000000
  1. Measure: cpu_load_short
  2. Tags: host and region
  3. Field value: value=0.64
  4. Timestamp: 1434055562000000000
